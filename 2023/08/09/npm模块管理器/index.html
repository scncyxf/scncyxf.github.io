npm有两个用途，一个是Node的开放式模块登记和管理系统，另一个是Node默认的模块管理器，用来安装和管理Node模块

npm不需要单独安装，在安装node的时候，会连带一起安装npm，但node附带的npm可能不是最新版本，最好用下面的命令，更新到最新版本

八 npm install

1 基本用法

Node模块采用npm install命令安装

每个模块可以全局安装，也可以本地安装。全局安装指的是将一个模块安装到系统目录中，每个项目都可以调用，一般全局安装只适用于工具模块。本地安装的是将一个模块下载到当前项目的node_modules子目录中，只有在项目目录中才能调用这个模块

2 安装不同版本

npm install命令总是安装模块最新版本，若要安装特定版本，可以在模块名后面加上@版本号

若使用–save-exact参数，会在package.json文件中指定模块的确切版本

npm install命令可以使用不同的参数，指定所安装的模块属于哪一种性质的依赖关系，即出现在package.json的哪一项中

npm install默认会安装dependencies字段和devDependencies字段的所有模块，若使用--production参数，则指安装dependencies字段的模块

3 避免系统权限

默认情况下，npm全局模块都安装在系统目录，需要root权限。我们也可以在不需要root权限的情况下安装全局模块

首先在主目录下新建配置文件.npmrc，在该文件中定义prefix变量

然后在主目录新建npm目录，全局安装模块都会安装到这个子目录，最后将这个路径加入PATH变量

4 package-lock.json文件

npm install的输入是package.json，它的输出是一颗node_modules树。为了在不同环境下生成相同的node_modules，npm使用package-lock.json文件来锁定版本

不同版本有不同的规则，在5.4.2版本后：

  • 若只有一个package.json文件，允许npm install会根据它生成一个package-lock.json文件

  • 若package文件的semver-range version和package-lock中版本兼容，即使package中有新版本，执行npm install会根据package-lock下载模块

  • 若手动修改了package的version ranges，且和package-lock版本不兼容，执行npm install时，package-lock将会更新到兼容package的版本

十二 其他命令

  • npm bin 显示模块执行目录

  • npm adduser 用于在npmjs.com注册一个用户

  • npm publish 发布模块到npmjs.com

  • npm login 登陆到npmjs.com

  • npm deprecate 废弃某个版本的模块

  • npm owner 用于管理模块维护者

  • npm home 打开模块的主页

  • npm repo 打开模块的代码仓库

  • npm outdated 检查当前项目所依赖的模块，是否已有新版本

  • npm prune 检查当前项目的node_modules目录中，是否有package.json没有提及到的模块

  • npm shrinkwrap 锁定当前项目的依赖模块版本，会生成一个npm-shrinkwarp.json文件夹
